# Frozen Application Policy Package v3.0

The authoritative application contract is defined by:
- controlled vocabularies and normalized intents;
- deterministic policy rules and rules-only baseline;
- redaction, market-overlay, linkage and template policies;
- semantic constraints and rationale templates;
- ground truth for all 40 messages;
- safety assertions and component schemas.

The active immutable bundle is identified by `configuration_manifest.json`. UI changes must create a draft version, pass component-schema validation, behavior fixtures, semantic validation and full regression before activation.
